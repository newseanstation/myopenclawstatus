# 新任务建立规则（统一格式）

以后新增任务，请按下面结构填写，便于快速自动创建与维护。

## 任务创建卡（复制此段）

- 任务名：
- 宠物名（可选）：
- Emoji（可选）：
- 任务目标（一句话）：
- 触发时间（cron/一次性）：
- 时区（默认 America/Toronto）：
- 输入来源（网站/文件/API）：
- 执行步骤（3-8条）：
- 输出格式（文本/Markdown/PDF/图片）：
- 投递渠道（Telegram/其他）：
- 投递对象（chat id / 用户）：
- 失败兜底策略（必须）：
- 是否允许补发（是/否）：
- 优先级（高/中/低）：

## 规则说明

1. 先定义“最小可交付”
   - 即使失败，也要保证至少发出一条有价值的文本结果。
2. 再定义“增强交付”
   - 如 PDF/附件/多来源扩展，失败不阻断主结果。
3. 来源控制
   - 初期优先可访问、可验证的 5-12 个来源，后续再扩展。
4. 输出可读
   - 每条信息控制在约 3-6 句，附原始链接。
5. 可维护
   - 任务改动优先改模板文件，减少频繁改指令本体。
